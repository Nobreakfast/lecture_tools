# 课程助手教师使用说明

这份文档面向教师与管理员，介绍如何使用教师面板完成建课、发题、查看答题、发布资料、收作业和生成总结。文档页与教师操作页分离，适合在教学现场单独打开参考。

> 建议使用方式：在教师面板右上角点击“使用文档”，新窗口保留说明页，原窗口继续操作。
>
> 本文档由 AI 生成，如教程无法解决问题，则请联系赵浩诚老师 `18550215939`。

## 1. 登录与进入教师面板

1. 打开教师入口 `/teacher` 或 `/t`。
2. 输入工号和密码后登录。
3. 登录成功后，会进入教师面板，并在顶部看到当前教师姓名与课程切换栏。

![教师面板首页：顶部可切换课程，右上角可打开使用文档]( /static/docs/screenshots/teacher-dashboard.png )

## 2. 创建课程与发给学生的邀请码

在“课程”页签中，可以完成课程初始化。

1. 在“课程名称”里填写中文课程名，例如“机器学习导论”。
2. 在“英文名称”里填写英文或内部名称，例如 `Machine Learning Intro`。
3. 点击“创建课程”。
4. 创建完成后，课程会出现在“我的课程”和顶部课程切换栏中。
5. 每门课程都会生成独立邀请码，可直接复制邀请码、复制加入链接，或下载二维码发给学生。

教师使用建议：

- 中文名称适合展示给学生与教师本人查看。
- 英文名称会作为课程内部目录名的一部分，建议简洁稳定。
- 如果一位教师管理多门课，请先从顶部切换到目标课程，再进行上传或发布操作。

![课程页：创建课程后可复制邀请码、加入链接和二维码]( /static/docs/screenshots/teacher-course-and-invite.png )

## 3. 上传并加载小测题库

在“上传”页签下，默认会进入“小测题库”子页。

常见工作流如下：

1. 先在顶部课程栏中选中目标课程。
2. 点击“选择文件”，上传一个或多个 YAML 题库文件。
3. 上传成功后，题库会出现在“已上传题库”列表中。
4. 点击对应题库后的“加载”，把它设为当前课堂使用的题库。
5. 如需重新编辑，也可以点击“编辑”进入可视化题库编辑页。

如果你的题库依赖图片等资源，建议使用“上传题库包（YAML + 图片等）”方式，保持题目和配图一起管理。

![题库上传页：上传 YAML 后，可直接加载为当前小测]( /static/docs/screenshots/teacher-quiz-upload.png )

## 4. 新建小测题目（可视化编辑）

如果你不想手写 YAML，也可以直接在教师端新建和编辑题目。

进入方式：

1. 打开“上传”页签中的“小测题库”子页。
2. 点击顶部的“+ 新建题库”。
3. 输入题库 ID 和标题后进入题库编辑器。

进入编辑器后，可以直接完成以下工作：

- 在表单视图中添加单选题、多选题、判断题、问卷题和简答题。
- 为每道题设置题目 ID、题干、选项、正确答案。
- 为简答题选择回答方式，例如 `text`、`image`、`code`、`text_image`。
- 在“高级选项”中补充解析、知识点标签、题池标签和参考答案。
- 为题干或选项上传配图。
- 通过拖拽题号调整题目顺序。
- 使用“预览”检查学生端效果，使用“校验”检查格式问题。
- 使用“保存到题库”保存，也可以依靠页面自动保存。

编辑建议：

- 题目 ID 建议保持简洁且唯一，例如 `q1`、`q2`。
- 保存前最好先点一次“校验”，可以提前发现缺少答案、重复 ID 等问题。
- 如果是课堂现场快速出题，优先使用表单视图；如果需要细调 YAML 结构，再切换到 YAML 视图。

![题库编辑器：可通过表单视图新增题目、设置答案、标签与高级选项]( /static/docs/screenshots/teacher-quiz-editor.png )

## 5. 使用 AI 辅助生成小测题目

题库编辑器内置了两个 AI 能力，适合用来提速出题和补全文案。

### AI 生成

“AI 生成”适合从零开始生成一批题目。

推荐流程：

1. 进入题库编辑器后，点击顶部“AI 生成”。
2. 在弹窗中输入清晰描述，例如：
3. 写明主题、题型、题量、难度、选项数量，必要时写明课程背景。
4. 点击“生成”，等待 AI 返回 YAML 结果。
5. 先看预览，再选择“替换当前内容”或“追加到末尾”。
6. 应用后，回到表单视图检查题目，再手动调整题干、答案与解析。

提示词示例：

- `生成5道关于 Python 基础语法的单选题，难度中等，每题4个选项，并给出答案和解析。`
- `为机器学习导论课程生成3道判断题和2道简答题，聚焦过拟合与正则化。`
- `生成适合大一学生的凸优化入门小测，题目尽量贴近课堂案例。`

### AI 补全

“AI 补全”适合已经有题目，但缺少解析或知识点标签时使用。

它会在现有题目的基础上，尝试补充：

- 解析
- 知识点标签

适用场景：

- 题目已经写好，但来不及逐题补解析。
- 已经有答案，希望 AI 帮你统一补齐标签与说明文字。

使用建议：

- AI 生成后一定要人工复核，尤其是正确答案、解析准确性和措辞是否符合你的课程。
- 如果 AI 返回格式异常，可以切到 YAML 视图手动修正。
- 对专业性较强的课程，提示词里尽量加入课程名、章节名、难度和学生层次，效果通常更稳定。

![AI 生成面板：可输入提示词，生成并替换或追加题目内容]( /static/docs/screenshots/teacher-quiz-ai.png )

## 6. 开关答题入口与查看答题情况

切换到“答题”页签后，可以控制课堂答题流程。

这里会显示：

- 当前已加载题库名称
- 入口当前是“已开放”还是“已关闭”
- 已进入人数
- 已提交人数
- 当前课程下的答题记录列表

常见操作：

1. 点击“开启入口”，允许学生进入本课程当前小测。
2. 学生开始答题后，顶部人数会实时变化。
3. 当需要截止作答时，点击“关闭入口”。
4. 点击“查看”可以查看单个学生的作答详情。
5. 点击“导出 CSV”可以导出答题结果。
6. 如果当前题库测试数据需要清理，可以使用“清空答题”。

课堂建议：

- 上课前先确认“当前题库”是否正确。
- 如果已经切换课程，建议重新看一眼“答题”页，避免误开到其他课程。
- 使用“筛选题库”可查看历史题库或全部题库的答题记录。

![答题页：可开启入口、实时查看进入人数和提交人数，并查看学生记录]( /static/docs/screenshots/teacher-attempts.png )

## 7. 上传课程资料给学生

在“上传”页签中切换到“课程资料”子页，可以管理资料文件。

支持的典型用法：

1. 上传 PDF 讲义、代码样例、实验说明或其他课程材料。
2. 上传后，材料会出现在下方列表。
3. 对 PDF 文件可直接点击“预览”。
4. 对任意材料可控制“显示 / 隐藏”，决定学生端是否可见。
5. 如需修正文件名，可使用“改名”；如需移除旧版本，可使用“删除”。

适合上传的资料包括：

- 讲义 PDF
- 示例代码压缩包
- 数据集说明
- 课堂补充说明文档

![课程资料页：可上传讲义、代码或其他材料，并控制学生是否可见]( /static/docs/screenshots/teacher-materials.png )

## 8. 发布作业与收取学生提交

在“上传”页签中切换到“作业”子页，可以完成作业布置与收取。

建议流程：

1. 在“作业编号”中输入本次作业标识，例如 `task_1` 或 `week3_hw`。
2. 上传作业说明文件。
3. 发布后，下方“已发布作业”中会列出该作业及其文件。
4. 通过“显示 / 隐藏”控制学生端是否可见。
5. 学生提交后，可在“学生提交”区域按作业筛选查看。
6. 如学生上传了多个文件类型，教师可分别下载，也可以下载“全部”归档包。

使用建议：

- 作业编号一旦开始发给学生，尽量不要频繁变动。
- 如果要更新作业说明，建议直接上传新文件并检查显示状态。
- 批量下载前，先在筛选框中选定作业编号。

![作业页：可发布作业文件，并查看学生提交和下载归档]( /static/docs/screenshots/teacher-homework.png )

## 9. 生成 AI 总结与历史趋势分析

在“其它”页签中，可以生成两类总结：

1. 当前题库总结：基于当前加载题库和已提交答题，生成错题分析、学生反馈总结和教学建议。
2. 历史趋势总结：纵向分析本课程多个小测的表现变化，帮助教师了解长期趋势。

推荐用法：

1. 等学生提交完成后，再点击“生成 AI 总结”。
2. 如果修改了答题数据，可点击“刷新”重新读取。
3. 生成后可以直接“复制总结”或“下载 .md”。
4. 如果 AI 服务未配置或暂时不可用，页面会提示失败信息，此时不影响其他教师功能使用。

![其它页：可生成当前题库 AI 总结和历史趋势总结]( /static/docs/screenshots/teacher-summary.png )

## 10. MCP 智能助手工具

在“其它”页签的下方，可以管理 MCP 智能助手配置。

它适合以下场景：

- 用自然语言查询课程数据
- 在 Trae、Claude Desktop、Cursor 中连接课程助手服务
- 快速检索作业、题库和答题相关信息

配置前请确认：

1. 打开“启用长效 MCP Token”开关。
2. 直接复制页面中的配置示例，粘贴到客户端的 MCP 配置中。
3. 如果服务器地址变化，需要重新复制最新配置。
4. 如果关闭开关，之前复制出去的 token 会立即失效。

当前 MCP 已提供的工具（tools）包括：

- `list_courses`：列出当前教师的课程列表（含课程 ID、标识、邀请码）
- `get_quiz_attempts`：查询课程内学生小测记录（按“同一学生”保留最佳成绩）
- `get_summary_stats`：查询课程当前加载题库的统计概览（人数、均分、每人正确数/总数）
- `get_quiz_feedback`：查询单次小测的反馈汇总（问卷题选项分布 + 简答题高频反馈）
- `get_quiz_question_stats`：查询单次小测逐题统计（正确率、作答数、常见错误选项）
- `get_homework_submissions`：查询课程作业提交情况（可按作业编号筛选）

常用提问示例（直接对智能助手说）：

- “列出我所有课程，并告诉我这周要用哪个邀请码。”
- “帮我看一下课程 3 这次小测的逐题正确率最低的 10 题。”
- “课程 3 的小测里，问卷题（检查题）反馈是什么分布？大家主要写了哪些建议？”
- “课程 3 的作业 task_1 谁还没交报告？谁只交了代码？”

## 11. 修改密码与退出登录

教师页右上角提供两个常用入口：

- “修改密码”：用于更新当前账号密码。
- “退出登录”：退出当前教师会话。

建议首次拿到账号后立即修改为个人密码，并妥善保存。

## 12. 常见课堂操作顺序

如果你是第一次完整使用本系统，可以按下面顺序操作：

1. 登录教师面板。
2. 创建课程。
3. 复制邀请码或链接发给学生。
4. 上传并加载本次小测题库。
5. 进入“答题”页确认题库正确后，开启入口。
6. 上课过程中观察已进入与已提交人数。
7. 截止后关闭入口。
8. 导出 CSV，或进入“其它”页生成 AI 总结。
9. 如有作业，再到“作业”子页发布说明并查看学生提交。

## 13. 常见问题

### 登录后看不到课程怎么办

先确认当前账号是否正确，以及课程是否由这个教师账号创建。课程数据按教师隔离，不同教师之间默认互不可见。

### 学生说进不去小测怎么办

优先检查以下几项：

1. 是否已经选中了正确课程。
2. 是否已经加载了题库。
3. “答题”页中的入口是否已经开启。
4. 学生使用的邀请码是否属于这门课。

### AI 总结按钮点击后没有结果怎么办

通常先检查 AI 服务配置是否可用。如果只是 AI 不可用，课程、题库、资料、作业与答题等核心功能仍可继续使用。

### 文档页打不开怎么办

请先确认自己仍处于登录状态。如果教师会话已过期，重新登录后再从教师面板右上角点击“使用文档”即可。
