# 《最优化方法》智能体可交互数学可视化 Roadmap

## 目标定位

面向《最优化方法》课程，把现有课程助手升级为“智能体驱动的可交互数学实验室”。学生提出数学概念、算法过程或作业理解问题时，系统不只返回文字解释，而是生成可验证的二维/三维可视化场景、动画演示或交互控件，帮助学生直观理解优化算法。

参赛建议主报 T1「AIGC·智能体技术与教育教学创新应用」，以 3D 坐标系、优化轨迹和算法交互演示作为跨赛项亮点。核心叙事是：智能体把抽象数学问题转化为可看、可调、可验证、可追问的学习对象。

## 设计原则

- Agent 不直接自由生成给学生运行的代码；Agent 生成受约束的 scene spec。
- 数学计算由系统内置的确定性内核完成，避免模型心算或随意推导。
- 前端只渲染白名单模板，模板经过教师审核和测试。
- 对学生展示的内容标注可靠性来源：已验证计算、课程资料依据、教学演示、需要教师确认。
- 所有生成过程保留可审计轨迹，错误反馈进入 Q&A 和教师复核闭环。

## Roadmap

### Phase 0：参赛 Scope 锁定

交付物：
- 明确作品名称、课程章节和演示主线。
- 选定 3-5 个最能体现《最优化方法》魅力的场景。
- 准备一套真实课堂前后测或学生反馈采集方案。

建议演示主线：
- 梯度下降如何沿负梯度方向下降。
- 学习率过小、合适、过大时的轨迹差异。
- 非凸函数中的局部最优、鞍点和初始点敏感性。
- 约束优化中的可行域、等高线相切和 KKT 直观解释。

### Phase 1：可验证数学场景引擎

交付物：
- 定义 scene spec JSON schema。
- 实现白名单模板注册表。
- 实现数学内核：函数解析、梯度计算、迭代轨迹、数值边界检查。
- 为每个模板增加基准测试样例。

首批模板：
- `gradient_descent_2d`：二维等高线、梯度向量、迭代轨迹、loss 曲线。
- `learning_rate_compare`：多个学习率的收敛/震荡/发散对比。
- `newton_method_2d`：二阶近似、牛顿方向、收敛轨迹。
- `constrained_optimization_2d`：可行域、等高线、边界最优点。
- `kkt_geometry`：活跃约束、法向量、互补松弛的几何说明。

### Phase 2：学生 Agent 到交互演示的闭环

交付物：
- 学生提问分类：概念解释、算法演示、作业提示、证明思路、需教师确认。
- Agent 输出 scene spec 草稿，而不是任意代码。
- scene spec 经过校验后生成学生可见演示。
- 学生可以调参数：初始点、学习率、迭代次数、显示维度。
- 演示旁边展示“解释 + 检查步骤 + 继续追问入口”。

关键用户故事：
- 作为学生，我问“为什么学习率太大会震荡”，系统生成同一函数在不同学习率下的轨迹对比。
- 作为学生，我问“拉格朗日乘子为什么是相切”，系统显示约束曲线和目标函数等高线，并允许拖动约束。
- 作为学生，我问“这个函数是不是凸函数”，系统给出局部可视化和判断边界，不把局部图像误说成全局证明。

### Phase 3：准确性、安全和教师复核

交付物：
- scene spec 校验器：函数合法性、参数范围、维度匹配、数值稳定性。
- 回答可靠性标签：已验证计算、基于课程资料、教学演示、需要教师确认。
- 教师纠错入口：一键把错误解释转为 Q&A 或课程 FAQ。
- 红队测试集：隐藏资料诱导、套取作业答案、错误数学结论、越权读取数据。
- Trajectory 脱敏与保留策略：学生身份、原始提问和模型输出分级保存。

### Phase 4：教学效果证据和比赛材料

交付物：
- 课堂演示视频：2-5 分钟，展示学生提问到交互演示生成的全过程。
- 项目报告：背景、技术路线、教学设计、创新点、应用效果、风险控制。
- 对比数据：使用前后学生对梯度、步长、KKT 等概念的理解变化。
- 学生反馈：哪些可视化最有帮助，哪些解释仍需教师介入。
- 源码片段和系统架构图：展示 Agent、数学内核、模板渲染、教师复核闭环。

## 验收标准

- 学生能在 10 秒级别得到至少一个可交互优化算法演示。
- 同一问题重复生成时，scene spec 稳定且可复现。
- 每个首批模板至少有 3 个固定测试样例。
- 学生端不会展示隐藏资料、教师数据、MCP token 或其他学生数据。
- 教师能看到生成轨迹、纠错并沉淀为 FAQ。
- 项目报告能清楚对应 T1 评分点：创意创新、技术实现、实用价值、报告文档、现场演示。
