# ADR-0001 技术选型

## 决策
- 后端使用 Go + SQLite
- 前端使用内嵌静态页面（无框架、零构建）
- 实时数据使用 SSE
- 题库格式使用 YAML
- AI 能力采用“可选配置”的外部 HTTP 适配（不配置则退化为内置规则总结）
- 课件采用文件系统目录管理（PDF）

## 原因
- Go 可编译单文件，适配本机启动与 CLI 场景
- SQLite 无需单独服务，便于课堂部署
- SSE 实现简单，满足管理员实时看板
- YAML 便于教师手工维护题库
- 外部 AI 接入保持可插拔，避免强依赖特定厂商
- PDF 直接落盘便于离线课堂与统一分发

## 影响
- 后续可平滑迁移到 Vue 前端与 PostgreSQL
- 当前实现优先保证稳定与可运维性
